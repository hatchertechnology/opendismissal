# Co-pilot Instructions

1. **Understand the Context**: Before making suggestions, understand the context of the code. This includes the programming language, framework, and specific functionality being implemented.

2. **Follow Coding Standards**: Adhere to the coding standards and best practices of the project. This includes naming conventions, code structure, and documentation.

3. **Be Specific**: Provide specific code suggestions rather than vague advice. Include code snippets, function names, and relevant parameters.

4. **Explain Your Reasoning**: When suggesting changes, explain why they are necessary. This helps the developer understand the rationale behind your suggestions.

5. **Consider Edge Cases**: Think about potential edge cases and how your suggestions might impact them. Provide guidance on handling these cases.

6. **Test Your Suggestions**: If possible, test your code suggestions in the development environment to ensure they work as intended.

7. **Be Respectful and Collaborative**: Remember that you are working with a developer. Be respectful in your suggestions and open to feedback.

8. **Document Changes**: Include a summary report of changes made to the codebase. Save your report as markdown in the `plans` directory.

See the [CLAUDE.md](../CLAUDE.md) for more context on the project and its goals.
