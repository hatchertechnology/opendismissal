# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this folder.

## Folder Overview

This folder contains plans, reviews, and development notes for the OpenDismissal project, a Django-based school dismissal management system.

## Naming Conventions
- **Files Names**: 
  - Use lowercase with underscores (e.g., `claude.md`, `development_notes.md`)
  - Use descriptive names that reflect content (e.g., `feature_plan.md`, `development_report.md`)
  - Include versioning in filenames if applicable (e.g., `development_commands_v1.md`)
  - Never use spaces in filenames; use underscores instead
  - never use special characters or punctuation in filenames
  - Include authors names in filenames if relevant (e.g., `feature_john_doe.md`, `bugfix_jane_doe_review_john_doe.md`)
- **Folders**: 
  - Use lowercase with hyphens (e.g., `plans`, `reviews`)
  - Files that relate to the same topic should be grouped in the same folder. For example all files related to a feature like mvp should be in the same folder.
  - Use descriptive names that reflect content (e.g., `student_edit_ui`, `mobile_ui`, `dismissal_code_management`, etc.)
  
- **Content**: 
  - Use clear, descriptive titles and headings
  - Use bullet points or numbered lists for clarity
  - Include relevant links to related files or resources
  - Use consistent formatting for code snippets and commands
  - Use comments in code snippets to explain complex logic or important details
  - Use markdown syntax for formatting (e.g., `# Heading`, `## Subheading`, `**bold**`, `*italic*`)
  - Include examples to illustrate complex concepts or usage
  - Use a consistent tone and style throughout the document
  - Keep sentences concise and to the point
  - Avoid jargon and technical language where possible
  - Use active voice for clarity
  - Use proper grammar and spelling
  - Use headings and subheadings to organize content logically
  - Use tables for structured data where appropriate
  - Use links to reference external resources or documentation
  - Use images or diagrams to illustrate complex concepts or workflows. Mermaid diagrams are preferred for architecture and flowcharts.
  - Use code blocks for code snippets, ensuring proper syntax highlighting
  - Include enough context for other developers to be able to execute the work or debug issues in the future.
